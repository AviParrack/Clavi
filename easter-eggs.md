# Easter Eggs

*We leave things for the careful reader. Some are fun. Some are breadcrumbs for future historians. Some are infrastructure. Pitch new ones freely — the bar is: would a careful reader who finds this feel rewarded?*

---

## Conventions

- **Git as running commentary** — commits are informative, have personality, and are readable as parallel text; never "fix bug" or "update file"; future historians should find them interesting
- **State-of-mind comments** — honest code comments ("this was written at 2am and it shows" / "genuinely proud of this one")
- **Fun fact injections** — non-sequitur but delightful facts buried in comments, for the reader who notices
- **ASCII art** — headers, flourishes, flex moments in code files
- **References** — literary, lyrical, poetry, books, comics, TV; sprinkled when artful, never forced; rewards the well-read
- **Self-mirroring structure** — essays whose structure mirrors their content when the opportunity arises
